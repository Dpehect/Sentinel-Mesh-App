# Public reports

Public report pages are server rendered and designed for SEO/GEO discoverability. A production publication must contain:

- repository owner/name and immutable commit SHA;
- scanner names and versions;
- scan start/completion timestamps;
- score methodology version;
- finding counts and evidence links;
- attack-path explanation;
- explicit limitations and false-positive disclosure;
- canonical URL and JSON-LD metadata.

Private repositories must never become public without an explicit publication action from an authorized tenant member.
