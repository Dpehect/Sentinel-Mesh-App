# Open-source release checklist

- [ ] Run `npm run verify`
- [ ] Run `npm run test:e2e`
- [ ] Replace demo credentials and secrets
- [ ] Capture dashboard, findings and attack graph screenshots
- [ ] Record a 90-second demo
- [ ] Publish architecture and threat-model diagrams
- [ ] Enable branch protection and required quality checks
- [ ] Add SECURITY.md disclosure contact
- [ ] Tag v1.1.0 and publish release notes
