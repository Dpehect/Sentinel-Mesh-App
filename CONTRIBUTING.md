# Contributing

1. Create a focused branch and describe the security or product problem.
2. Add or update unit/integration fixtures for behavioral changes.
3. Run `npm run verify` and `npm run test:e2e`.
4. Do not include real secrets, exploit payloads against third parties or unauthorized scan targets.
5. Document architecture trade-offs for material changes.
6. Submit a pull request with scope, verification evidence and screenshots for UI changes.
